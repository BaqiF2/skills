# 项目规格说明制定任务

## 项目文档上下文

@CLAUDE.md 或 @.claude/CLAUDE.md

## 任务目标

你的任务是帮我为当前这个项目制定一份规格说明。

## 工作方法

使用"询问用户问题"工具，通过与我交流并收集有关项目实施、用户界面与用户体验、技术栈、顾虑、权衡等的详细信息，来协助制定 @.self_spec/ 中的规格说明。

## 关键原则

1. 务必确保所提出的问题并非显而易见，而应进一步深入探究潜在的需求和限制因素
2. 持续且系统地对我进行访谈，直至规范要求全部达成
3. 记录下所有的回答和见解，以形成一份全面且结构严谨的规范文件，作为项目的基础
4. 充分理解并参考上述 CLAUDE.md 中的项目架构、设计模式和技术约束

## 输出要求

为这次讨论的目标生成一个简短的标题（格式为：${YYYY-MM-DD-title}），在 .self_spec 生成该标题的文件夹，将文件放在该文件夹中，文件名称统一为 design.md
